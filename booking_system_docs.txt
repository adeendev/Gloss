## 🎯 System Overview

**Purpose:** Production-ready booking system for car detailing businesses with WhatsApp communication, payment processing, and multi-tenant support.

**Key Technologies:**
- **Frontend:** Next.js 14 (App Router), React, TypeScript, Tailwind CSS, Shadcn/ui
- **Backend:** Next.js API Routes
- **Database:** PostgreSQL + Prisma ORM
- **Auth:** NextAuth.js
- **Payments:** Stripe
- **Communication:** WhatsApp Business Cloud API
- **Deployment:** Vercel + Railway

### 1. Customer Booking Flow

**Route:** `/booking/[businessId]`

**Features:**
- Real-time availability calendar
- Service selection with pricing

- Customer information form
- Advance payment (20% default)
- WhatsApp confirmation

**Key Components:**
```typescript

```

### 2. Admin Dashboard

**Route:** `/admin/dashboard`

**Features:**
- Today's bookings overview
- Revenue tracking
- Upcoming appointments
- Payment status
- Quick actions

**Stats Displayed:**
- Total bookings (today/week/month)
- Revenue (collected/pending)
- Service breakdown
- Customer metrics

### 3. WhatsApp Integration

**Automated Notifications:**
1. **Booking Confirmation** - Immediately after booking
2. **24-Hour Reminder** - Day before appointment
3. **Completion Invoice** - After service completion
4. **Review Request** - 2 hours after completion

**Two-Way Chat:**
- Admin can chat with customers
- View conversation history
- Send template messages
- Message status tracking (sent/delivered/read)

### 4. Payment Processing

**Stripe Integration:**
- Card payments
- Payment intents
- Webhook handling
- Refund support

**Payment Flow:**
1. Customer selects service
2. 20% advance payment required
3. Payment processed (Stripe)
4. Booking confirmed
5. WhatsApp notification sent
6. Remaining 80% on completion

## 📱 WhatsApp Integration Guide

### Setup Steps

1. **Create Meta Business Account**
   - Go to business.facebook.com
   - Create business portfolio
   - Add WhatsApp Business Account

2. **Get API Credentials**
   - Go to developers.facebook.com
   - Create app → Business → WhatsApp
   - Get Phone Number ID
   - Get Access Token
   - Get Business Account ID

3. **Configure Webhooks**
   ```
   Callback URL: https://yourdomain.com/api/whatsapp/webhook
   Verify Token: [your-random-token]
   Fields: messages
   ```

4. **Create Message Templates**
   ```
   Template Name: booking_confirmation
   Category: TRANSACTIONAL
   Language: English
   
   Body:
   Hi {{1}}! Your {{2}} appointment is confirmed for {{3}} at {{4}}. 
   We've received your advance payment of {{5}}. 
   See you soon! - {{6}}
   ```

### WhatsApp Client Implementation

```typescript
// lib/whatsapp.ts
import axios from 'axios'

export class WhatsAppClient {
  private baseUrl = 'https://graph.facebook.com/v18.0'
  private phoneNumberId = process.env.WHATSAPP_PHONE_NUMBER_ID
  private accessToken = process.env.WHATSAPP_ACCESS_TOKEN

  async sendTextMessage(to: string, message: string) {
    const response = await axios.post(
      `${this.baseUrl}/${this.phoneNumberId}/messages`,
      {
        messaging_product: 'whatsapp',
        to,
        type: 'text',
        text: { body: message }
      },
      {
        headers: {
          'Authorization': `Bearer ${this.accessToken}`,
          'Content-Type': 'application/json'
        }
      }
    )
    return response.data
  }

  async sendTemplateMessage(to: string, templateName: string, params: string[]) {
    // Implementation for template messages
  }

  async markAsRead(messageId: string) {
    // Mark message as read
  }
}
```

## 💳 Payment Integration Guide

### Stripe Setup

```typescript
// lib/stripe.ts
import Stripe from 'stripe'

export const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
  apiVersion: '2023-10-16'
})

export async function createPaymentIntent(amount: number, bookingId: string) {
  return await stripe.paymentIntents.create({
    amount: Math.round(amount * 100), // Convert to cents
    currency: 'usd',
    metadata: { bookingId },
    automatic_payment_methods: { enabled: true }
  })
}
```

## 🎨 White-Label Customization

### Multi-Tenant Architecture

Each business gets:
- Unique booking URL: `/booking/[businessId]`
- Separate admin dashboard
- Isolated data (enforced by Prisma)
- Custom branding (logo, colors, name)

### Customization Options

```typescript
// BusinessSettings model
{
  // Branding
  businessName: string
  logo: string
  primaryColor: string
  secondaryColor: string
  
  // Working hours
  workingDays: ['MON', 'TUE', 'WED', 'THU', 'FRI']
  workingHoursStart: '09:00'
  workingHoursEnd: '18:00'
  
  // Booking rules
  slotDuration: 30 // minutes
  advanceBookingDays: 30
  minAdvanceHours: 2
  
  // Payments
  advancePaymentPercent: 20
  currency: 'USD'
  
  // Notifications
  sendWhatsApp: true
  sendEmail: false
}
```

### Adding New Business

```typescript
// Create business
const business = await prisma.business.create({
  data: {
    name: "Joe's Detailing",
    email: "joe@example.com",
    phone: "+1234567890",
    settings: {
      create: {
        advancePaymentPercent: 25,
        workingDays: ['MON', 'TUE', 'WED', 'THU', 'FRI', 'SAT']
      }
    }
  }
})

// Their booking URL: /booking/[business.id]
```

## 🔒 Security Best Practices

1. **Authentication:** NextAuth.js with secure session management
2. **Authorization:** Role-based access control (Admin/Staff)
3. **Data Validation:** Zod schemas on all API endpoints
4. **Payment Security:** Never store card details (use Stripe)
5. **Webhook Verification:** Verify all webhook signatures
6. **Rate Limiting:** Implement on public endpoints
7. **CORS:** Configure allowed origins
8. **Environment Variables:** Never commit secrets to git
